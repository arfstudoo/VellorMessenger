# VellorMessenger - Правила разработки

## Архитектура проекта

### Структура компонентов
- **App.tsx** - Главный компонент с состоянием приложения, темами, аудио/уведомлениями
- **components/** - UI компоненты (модальные окна, чаты, списки)
- **hooks/** - Кастомные хуки (useChatData для логики чатов)
- **types.ts** - TypeScript типы и интерфейсы
- **constants.ts** - Константы (звуки, моки)
- **supabaseClient.ts** - Клиент базы данных

### Технологический стек
- React 18 + TypeScript
- Supabase (БД, Auth, Storage, Realtime)
- Framer Motion (анимации)
- Tailwind CSS (стили)
- Lucide React (иконки)
- Capacitor (мобильные платформы)
- Electron (десктоп)

## Стандарты кода

### TypeScript
- Всегда использовать строгую типизацию
- Избегать `any`, использовать конкретные типы из types.ts
- Интерфейсы для props компонентов обязательны

### React компоненты
- Функциональные компоненты с хуками
- Использовать `React.FC<Props>` для типизации
- Мемоизация через `useCallback`, `useMemo` для оптимизации
- Refs для DOM элементов и мутабельных значений

### Стилизация
- Tailwind классы для базовых стилей
- CSS переменные для тем: `--bg`, `--panel`, `--accent`, `--text`, `--border`, `--glass`, `--msg-me`
- Glassmorphism: `backdrop-blur-*` + прозрачность
- Анимации через Framer Motion

### Состояние
- Локальное состояние через `useState`
- Refs для значений, не требующих ре-рендера
- Supabase Realtime для синхронизации данных

## Функциональные требования

### Приватность
- Поддержка настроек приватности: `privacy_phone`, `privacy_last_seen`, `privacy_avatar`, `privacy_calls`
- Значения: `'everybody' | 'contacts' | 'nobody'`
- Принцип взаимности: если пользователь скрывает свой статус, он не видит чужие

### Сообщения
- Типы: `text`, `image`, `audio`, `file`, `system`, `location`
- Поддержка реакций (emoji)
- Редактирование и удаление
- Закрепление (pin)
- Ответы (reply)
- Индикатор "печатает..."

### Группы
- Создание с участниками
- Администраторы групп
- Системные сообщения о действиях
- Описание группы
- Баннер и аватар

### Звонки
- Аудио и видео через WebRTC
- Проверка настроек приватности перед звонком
- История звонков (локальное хранилище)

### Темы оформления
- 7 предустановленных тем: crimson, ocean, cyber, gold, emerald, obsidian, sunset
- Динамическая смена через CSS переменные
- Wallpaper градиенты
- Пульсирующие эффекты (отключаемые)

### Уведомления
- Браузерные уведомления
- Push уведомления (Capacitor)
- Звуковые уведомления (5 вариантов)
- Отключение для отдельных чатов (mute)

### Админ панель
- Доступ через 7 тапов по логотипу + PIN 2077
- Broadcast сообщения
- Режим обслуживания (maintenance)
- Просмотр онлайн пользователей

## Безопасность

### Критические правила
- **НИКОГДА** не коммитить API ключи в код
- Использовать `.env` для секретов
- Валидация на клиенте И сервере
- Санитизация пользовательского ввода
- RLS (Row Level Security) в Supabase

### Аутентификация
- Supabase Auth для регистрации/входа
- Проверка сессии при загрузке
- Автоматический logout при истечении токена

## Производительность

### Оптимизация
- React.memo для всех компонентов списков (ChatItem, MessageItem)
- useMemo/useCallback для тяжелых вычислений
- Lazy loading изображений через LazyImage компонент
- Throttle для scroll events (100ms)
- Debounce для search и typing (500ms)
- Батчинг realtime обновлений (50ms)
- GPU acceleration через will-change и transform
- Intersection Observer для lazy loading
- Виртуализация списков (при >100 сообщений)
- Оптимизация изображений перед загрузкой
- Кэширование аватаров

### Realtime
- Подписка только на нужные каналы
- Отписка при размонтировании компонентов
- Батчинг обновлений состояния (50ms задержка)
- Один канал для нескольких событий где возможно

### Утилиты производительности
- `utils/performance.ts` - debounce, throttle, hooks
- `components/ui/LazyImage.tsx` - ленивая загрузка изображений
- CSS оптимизации в `index.css`

## Мобильная разработка

### Capacitor
- Push уведомления через `@capacitor/push-notifications`
- Обработка фона/переднего плана через `@capacitor/app`
- Проверка платформы: `Capacitor.getPlatform()`
- Сохранение токенов в Supabase

### Адаптивность
- Mobile-first подход
- Breakpoint: 768px (md)
- Touch-friendly элементы (минимум 44x44px)
- Swipe жесты для действий

## Специфичные правила VellorMessenger

### Цветовая схема
- Основной акцент: vellor-red (#ef4444, #ff0033)
- Темный фон: черный с прозрачностью
- Glassmorphism: backdrop-blur + rgba

### Анимации
- Framer Motion для всех переходов
- `initial`, `animate`, `exit` паттерн
- Smooth scroll для сообщений
- Active scale эффекты на кнопках

### Звуки
- Локальные файлы в `/sounds/`
- Fallback на CDN при ошибке
- AudioContext API для воспроизведения
- Проверка настроек перед воспроизведением

### Иконки
- Lucide React
- Размер по умолчанию: 20px
- Цвет: white/60 → white при hover
- Transition на все изменения

### Модальные окна
- Полноэкранные на мобильных
- Backdrop blur + черный overlay
- Анимация появления/исчезновения
- ESC для закрытия

### Чаты
- Сортировка: закрепленные → по времени последнего сообщения
- Папки: все, личные, группы
- Контекстное меню (правый клик)
- Индикаторы: непрочитанные, typing, online

### Сообщения
- Разделители по датам
- "Непрочитанные сообщения" separator
- Swipe для ответа
- Long press для контекстного меню
- Автоскролл к новым сообщениям

## Известные ограничения

1. Звонки не работают в группах
2. Stories не реализованы (только флаг)
3. Нет E2E шифрования
4. Ограничение на размер файлов (зависит от Supabase Storage)
5. Голосовые сообщения только webm формат
